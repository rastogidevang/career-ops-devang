# UX AUDIT — career-ops-ui v1.58.51 (senior UX-designer pass)

**Date:** 2026-05-19
**Auditor:** Claude (Sonnet 4.6), senior UX-designer hybrid
**Method:** heuristic + task-based against `https://career-ops.org/docs`. Read-only, no code edits.
**Baseline:** the prompt's "already-closed" ledger (61 items, v1.55.1 → v1.58.50) — none re-filed without live regression evidence. New findings only.

---

## Intended user & outcome (one paragraph, from the docs)

A working knowledge-worker running a parallel, structured job search against multiple ATSes. They want to **paste one job URL** and get back, in 1–2 minutes, an honest A–F rubric-scored report (with score-driven thresholds for "apply / refine / pass"); they want their `cv.md`, `data/applications.md`, and `modes/_profile.md` to stay as plain markdown on their laptop; they want to drive the actual LLM call from whichever AI coding CLI they already use (Claude Code · Codex · Gemini · OpenCode · Qwen · Copilot · Kimi); and they want the headless web-ui live-eval to work on **any one** of the Anthropic / Gemini / OpenAI / Qwen / OpenRouter API keys ("OR" model). The UI must make the documented journey easier, never alter or contradict it.

---

## Top-5 user-impact findings (executive summary)

1. **UX-A1 (HIGH / promise fidelity) — Deep research saved brief still meta-narration, no defensive warning.** The single highest-value flow's deliverable doesn't match the docs' "6-section company brief" promise. C-1 prompt-layer is parent-owned, but the web-ui has shipped no defensive UX to tell the user "this output is not the canonical brief". The user sees the model's stream-of-consciousness and may believe that IS the brief. **Recommend: defensive `looksLikeStructuredBrief()` warning ribbon.**

2. **UX-A2 (MEDIUM / promise fidelity) — `#/config → Modes` field-form is mostly raw-textarea, contradicting the docs' §Step-5 promise.** The docs describe "Target Roles / Adaptive Framing / Comp Targets" as repeatable labelled line-inputs (per Quick Start Step 5). Live: each section is rendered as a single `<textarea>` with explanatory text "This section isn't a simple list — editing as raw text to avoid data loss". The "field-form" UX promise from `qa/REGRESSION-FINAL.md §4` is only **partially** delivered.

3. **UX-A3 (MEDIUM / first-run / orientation) — No live "active-provider" chip on Dashboard.** The OR-model is the core trust mechanic ("works on any one key"). Health page shows it; Dashboard, where users start, does not. New users must visit Health to know which provider their next ⚡ Run live will use. **Recommend: small `.dash-chip` near hero CTAs: "Live evals: Anthropic claude-sonnet-4-6 ⚡" or "Manual prompt mode (no API key)".**

4. **UX-A4 (MEDIUM / a11y as UX) — Language picker buttons are 23–25 px tall (below WCAG 2.5.5 AAA / Apple HIG 44 px / Material 48 px target sizes).** 8 small buttons clustered in the sidebar bottom. Easy to mis-tap on touch / trackpad-with-tremor users. WCAG 2.5.5 is AAA; WCAG 2.5.8 (Target Size Minimum, AA from WCAG 2.2) requires **24×24 px** which they barely meet (23×47 — width OK, height marginal).

5. **UX-A5 (MEDIUM / promise fidelity / regression) — Help TOC scroll-spy not firing.** Spec §13 UX-D-K v1.58.45 closure says IntersectionObserver applies `.toc-current` on scroll. Live on v1.58.51: 18 H2s with proper ids, 18 TOC links, **0** ever get the highlight class after scrolling. User loses orientation in a long help page. *(Confirmed in `2026-05-19-REGRESSION-v1.58.51.md` as NEW-K1.)*

## What's genuinely good (balanced audit)

- **CSP unconditional** (v1.58.4) — full directive set, no `'unsafe-eval'`, no `'unsafe-inline'` for `script-src`. Reference-quality.
- **URL validation envelope** (NEW-2 v1.58.7) — `${T}` / `{{T}}` / `<%T%>` paired template tokens rejected, single `{normal}` accepted. Semantic precision.
- **Apply checklist as actual checklist** (M-8 v1.58.13 + UX-D-D v1.58.46) — promise → delivery.
- **Notifications drawer** (v1.58.34/35) — bell + drawer + journal + `aria-expanded` + `aria-controls` + Playwright 7-step contract. Best-in-class chrome.
- **Focus-ring on form fields** (M-1 v1.58.9) — brand-red 2 px outline only on `:focus-visible` (keyboard). WCAG 2.4.7 done with style.
- **Followup ISO validation** (BUG-001) — junk dates blocked client-side; zero network call; localized example date.
- **Toast detail in `<details>`** (U-4 v1.58.24) — endpoint context preserved without leaking at the surface.
- **Saved-card title↔date structural** (M-4 v1.58.11) — `display: flex; gap: 8px` + semantic `<time datetime>`. (Marred by UX-A6.)
- **i18n discipline** — 8 locales, parity gate, autodetect, persist, `<html lang>` updates. NEW-D1 closure for `pip.title` ×3 locales done within one cycle.
- **`#/cv` H1 + subtitle** (U-1 v1.58.21) — proper big H1, single-H1 invariant intact.
- **Tracker actions disabled @0 rows** (U-10 v1.58.30) with localized tooltip — small, smart.
- **`Open Scan` rename** (UX-D-C v1.58.47) — label honesty over marketing.
- **Fixture-profile banner** (UX-D-B v1.58.48) — proactive on Dashboard, not buried.
