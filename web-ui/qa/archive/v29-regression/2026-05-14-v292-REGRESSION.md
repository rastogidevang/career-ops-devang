## Warnings

**W-001 — browser-cache invalidation on deploy.** Initial live probe of `#/evaluate?url=…` and `#/config?tab=modes` returned `"404 — страница не найдена"`. Root cause: browser-cached old `api.js` + `router.js` from the pre-v1.28.1 deploy were still loading despite server being on v1.29.2. After hard reload with `?cb=v292` cache-buster, both routes resolved correctly.

**Operational impact:** Users who connect to a freshly-deployed v1.29.2 stand without doing a hard refresh will see pre-v1.28.1 behavior (404 on query-string routes) until they reload. The fix is real and shipped, but a deploy-time cache-bust mechanism (e.g., version-stamped asset URLs) would prevent the user-facing artifact.

**Recommended follow-up:** add version-hash query string to `<script src="…">` and `<link href="…">` in `public/index.html`, or set `Cache-Control: no-store` on `/public/js/*` during dev/staging. This is a separate hardening item — not a v1.29.2 blocker.
