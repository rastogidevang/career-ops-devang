## Blocker

### B-1. TOCTOU between explicit DNS lookup and fetch in auto-pipeline + preview

**Files:** [`server/lib/routes/auto-pipeline.mjs:73-112`](../../server/lib/routes/auto-pipeline.mjs#L73-L112), [`server/lib/routes/pipeline.mjs:60-100`](../../server/lib/routes/pipeline.mjs#L60-L100)

**Issue (refined after live verification of source).** The catch on line 83 / 79 is **intentional fail-open** per the inline comment at `pipeline.mjs:68-71` ("Failing closed here would break test stubs … for no security gain"). The author's reasoning is sound: when DNS lookup throws, `fetch()` will do its own lookup and fail the network call.

The **actual vulnerability** is the time-of-check-to-time-of-use gap. The explicit `dnsLookup()` returns address #1; then `fetch()` does its **own independent lookup** that can return address #2. A DNS rebind attacker controls a hostname with TTL=0 and returns:

1. First lookup → public IP (passes `isPrivateOrLoopbackHost`).
2. Second lookup (inside `fetch()`) → `169.254.169.254`, `127.0.0.1`, or a LAN address.

The catch is irrelevant. The two-lookup pattern is the bug.

**Severity calibration:** under default `HOST=127.0.0.1` the attacker needs to trick the user into clicking a malicious URL, which requires social engineering. Under `HOST=0.0.0.0` (LAN deploy) any LAN peer can POST. Calling **blocker for LAN, high for loopback.**

**Fix.** Resolve once, then pin the socket to the pre-validated address. Two equivalent implementations:

```js
// Option A — undici Agent with custom lookup that returns the cached address
import { Agent } from 'undici';

async function safeFetch(url, opts = {}) {
  const host = new URL(url).hostname;
  const { address, family } = await dnsLookup(host, { verbatim: true });
  if (isPrivateOrLoopbackHost(address)) {
    throw new Error(`host resolves to private address: ${address}`);
  }
  const agent = new Agent({
    connect: { lookup: (_, __, cb) => cb(null, address, family) },
  });
  return fetch(url, { ...opts, dispatcher: agent });
}
```

```js
// Option B — explicit net.connect to the pre-validated IP, manual HTTP framing.
//           Heavier but avoids adding undici as a runtime dep.
```

**Pick A.** `undici` ships with Node ≥ 18 already (it's the underlying fetch implementation). The Agent is a no-cost wrapper.

**Test:** [`tests/ssrf-redirect-rebind.test.mjs`](../../tests/) — stub `dnsLookup` to return public IP on call #1 and `127.0.0.1` on call #2; assert the fetch fails (or the socket connects to the pinned IP, not 127.0.0.1).

**Effort.** ~40 LOC across both routes + 1 test, ~3 h.

### H-4. Path-traversal sanitization is inconsistent across 7 routes

**Files (verified by grep — exact line numbers):**

```
server/lib/routes/jds.mjs:31, 40              .replace(/[^\w\-.]/g, '')
server/lib/routes/content.mjs:321             .replace(/[^\w\-.]/g, '')
server/lib/routes/reports.mjs:45              .replace(/[^\w\-.]/g, '')   ← inconsistent w/ own sanitizeSlug at L29
server/lib/routes/runners.mjs:162, 201, 222   .replace(/[^\w\-.]/g, '')
server/lib/routes/llm.mjs:233, 241            .replace(/[^\w\-.]/g, '')
```

**Issue.** The simple regex keeps `.`, so `..pdf`, `....md`, leading-dot names all survive. Only [`reports.mjs:29-37`](../../server/lib/routes/reports.mjs#L29-L37) does the right thing:

```js
function sanitizeSlug(s) {
  // Strip non-[\w-.] AND any leading dots / dot-runs (path-traversal guard).
  return String(s || '')
    .replace(/[^\w\-.]/g, '')
    .replace(/\.md$/, '')
    .replace(/^\.+/, '')     // no leading "." or ".." prefixes
    .replace(/\.{2,}/g, '.') // collapse internal "..."
    .slice(0, 200);
}
```

And even `reports.mjs` doesn't use it everywhere — line 45 (`GET /api/reports/:slug`) uses the bad regex; line 59 (`POST` path) uses the good `sanitizeSlug`.

There is no `/` allowed (so escaping `output/` is impossible), but the broken pattern is duplicated **10 times** and is a recipe for future regression — one wrong copy-paste lets `../../etc/passwd` through.

**Fix.**

1. Move `sanitizeSlug` (rename to `sanitizePathName`) from `reports.mjs` to [`server/lib/security.mjs`](../../server/lib/security.mjs) and export it.
2. Drop the `.replace(/\.md$/, '')` — that's reports-specific suffix handling, not sanitization. Callers can strip suffixes themselves.
3. Replace all 10 call sites with `sanitizePathName(req.params.name)`.
4. Reject empty results (`if (!safe) return res.status(400).json({ ok: false, error: 'invalid name' });`).

**Test.** New [`tests/path-traversal.test.mjs`](../../tests/) that sweeps `..pdf`, `....md`, `..`, `.`, `/etc/passwd`, `foo/bar`, `foo\\..\\bar`, leading-dot, NUL bytes, URL-encoded `%2e%2e` against every endpoint accepting a `:name` / `:slug` / `?slug=` parameter. Expect 400 or 404 on every malicious input.

**Effort.** ~30 LOC consolidate + ~80 LOC test, ~1 h.

### H-5. No rate limit on LLM / auto-pipeline endpoints when `HOST=0.0.0.0`

**Files:** [`server/lib/routes/llm.mjs`](../../server/lib/routes/llm.mjs), [`server/lib/routes/auto-pipeline.mjs`](../../server/lib/routes/auto-pipeline.mjs)

**Issue.** `/api/evaluate`, `/api/deep`, `/api/mode/:slug`, `/api/auto-pipeline` all call Anthropic with `maxTokens: 8192`. Each call is roughly $0.05 against the user's `ANTHROPIC_API_KEY`. There is no per-IP / per-key / global throttle. Default `HOST=127.0.0.1` makes this academic — only the local user hits the endpoint. But `HOST=0.0.0.0` is supported (see CSP comment in `server/index.mjs`), and a LAN attacker can drain the key in minutes.

**Severity:** **high** when LAN-exposed, **low** on default loopback. Documented limitation in `docs/PRODUCTION-READINESS.md` says "ready for single-tenant loopback; LAN exposure waits on the v2.0 P-12 auth gate." This finding suggests the LAN gate should land *before* P-12 as a cheap interim defense.

**Fix.** In-memory token bucket, gated by `isPubliclyExposed()`:

```js
// server/lib/rate-limit.mjs (new)
import { isPubliclyExposed } from './security.mjs';

const BUCKETS = new Map(); // ip → { count, resetAt }
const LIMIT = 10;
const WINDOW_MS = 60_000;

export function llmRateLimit(req, res, next) {
  if (!isPubliclyExposed()) return next();
  const ip = req.ip;
  const now = Date.now();
  const b = BUCKETS.get(ip);
  if (!b || b.resetAt < now) {
    BUCKETS.set(ip, { count: 1, resetAt: now + WINDOW_MS });
    return next();
  }
  if (b.count >= LIMIT) {
    res.setHeader('Retry-After', Math.ceil((b.resetAt - now) / 1000));
    return res.status(429).json({ ok: false, error: 'rate limit: 10 req/min' });
  }
  b.count += 1;
  next();
}
```

Wire it into all four LLM-calling routes via `app.post('/api/evaluate', llmRateLimit, …)`.

**Test.** [`tests/rate-limit.test.mjs`](../../tests/) — set `HOST=0.0.0.0`, burst 11 requests, expect `10 × 2xx + 1 × 429` and the `Retry-After` header.

**Effort.** ~50 LOC middleware + 4-route wiring + ~30 LOC test, ~2 h.

### H-6. Concurrent-write race on `applications.md` + `pipeline.md`

**Files (verified live):**

[`server/lib/routes/tracker.mjs:42-75`](../../server/lib/routes/tracker.mjs#L42-L75):

```js
let content = '';
try { content = readFileSync(PATHS.applications, 'utf8'); } catch { content = ''; }
const existing = parseApplications(content);
// … dedup check …
const nextNum = String((Math.max(0, ...existing.map(r => parseInt(r.num, 10) || 0))) + 1).padStart(3, '0');
// … build row …
writeFileSync(PATHS.applications, updated);
```

Same pattern in [`pipeline.mjs:36-47, 121-135`](../../server/lib/routes/pipeline.mjs) and [`auto-pipeline.mjs:299-340`](../../server/lib/routes/auto-pipeline.mjs).

**Issue.** Read → compute → write with no locking. Two concurrent `POST /api/tracker` (e.g. user double-clicks Save, or auto-pipeline races with a manual add) both read `num=42`, both write `num=43`. **The earlier row is silently dropped.** Same race on pipeline-URL append (less catastrophic: dedup uses URL content, but two concurrent unique appends still race the file).

The dedup check on line 46 doesn't help — both racers see the same pre-dedup state.

**Fix.** Per-file async mutex. ~15 LOC of new code:

```js
// server/lib/file-lock.mjs (new)
const LOCKS = new Map(); // path → tail promise chain

export async function withFileLock(path, fn) {
  const prev = LOCKS.get(path) || Promise.resolve();
  let release;
  const next = new Promise((r) => { release = r; });
  LOCKS.set(path, prev.then(() => next));
  await prev;
  try {
    return await fn();
  } finally {
    release();
    if (LOCKS.get(path) === next) LOCKS.delete(path);
  }
}
```

Wrap every read-then-write at known paths:

```js
// tracker.mjs
import { withFileLock } from '../file-lock.mjs';
import { PATHS } from '../paths.mjs';

app.post('/api/tracker', async (req, res) => {
  await withFileLock(PATHS.applications, async () => {
    // existing logic
  });
});
```

**Test.** [`tests/concurrent-tracker-write.test.mjs`](../../tests/) — fire 20 concurrent `POST /api/tracker` with distinct `Acme #N` company names, then re-read the file and assert all 20 rows present with sequential `001` … `020` numbers.

**Effort.** ~25 LOC mutex + 3 call-site wraps + ~40 LOC test, ~1.5 h.

### v1.20.1 (target: 2026-05-14)

Order chosen so shared infrastructure lands first:

1. **H-3** (i18n coverage test + missing keys) — biggest WCAG win, no dependencies.
2. **H-4** (consolidate `sanitizePathName` in security.mjs) — touches `security.mjs`, blocks the rest from importing the helper.
3. **B-1** (TOCTOU SSRF fix) — uses the new pinned-lookup pattern; lands the highest-severity fix.
4. **H-6** (file mutex) — adds `server/lib/file-lock.mjs`; subsequent route fixes wrap.
5. **H-5** (rate-limit middleware) — adds `server/lib/rate-limit.mjs`; wires into 4 LLM routes.
6. **H-1 + H-2** (batch.js a11y + a11y-form-wires.test.mjs) — small, bundled together.
7. **CHANGELOG.md v1.20.1 entry** + locale stop-gaps + REVIEW doc + version bump + tag.

Each commit follows the convention `fix(<scope>): <ID> <one-line>` with the finding ID in the body. Example:

```
fix(security): B-1 — pin DNS lookup result via undici Agent

Auto-pipeline and pipeline-preview previously did an explicit
dnsLookup, then let fetch() do its own independent lookup,
opening a DNS-rebind TOCTOU window. Now we resolve once and
pin the socket via an undici Agent with a custom `lookup`
that returns the cached address.

Closes B-1.
```
