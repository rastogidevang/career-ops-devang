## Notable design decisions

### Sunset-window-now-closed (not just deprecated)

The user explicitly asked to merge `scan-en` + `scan-ru` into a
single `/api/stream/scan`. The consolidated endpoint had been live
since v1.12.0, and the EN/RU split aliases had RFC 8594 Sunset
headers since v1.15.0 telling external consumers about a 2026-10-01
removal window. v1.18.0 advances that removal to **now** — old paths
404, external integrations on those paths must migrate to
`?source=ats` or `?source=regional`.

The 404 is intentional. The alternative (silent route to SPA
catch-all returning index.html with 200) would mask the breakage and
leave external integrations broken-but-not-erroring. A clean 404
gives consumers a clear failure signal to investigate.

### WCAG 2.2 AA pass — what we did vs. what's still open

Done:
- **2.4.1 Bypass Blocks** — Skip to main content link.
- **2.4.7 Focus Visible** — global `*:focus-visible` style, high-contrast
  ring on modal close.
- **2.5.5 Target Size** (Level AAA, applied as AA-baseline) — skip
  link 44×44 minimum, btn-sm 32 px (uses the spaced-target AAA
  exception for compact table-row controls).
- **3.1.1 Language of Page** — `<html lang="en">` (JS bootstrap
  overrides on load to match user locale, but the SSR default now
  matches the SPA's default locale instead of the dev's locale).
- **1.3.1 Info & Relationships** — `#content tabindex="-1"` so skip
  link target focuses cleanly. ARIA roles + focus-trap were already
  shipped in v1.17.

Deferred (v1.19+):
- **1.4.3 Contrast (Minimum)** — text 4.5:1, large text 3:1. Need to
  audit every text-on-background pair across light + dark palettes
  via Chrome DevTools / axe.
- **2.5.5 per-component** — verify every interactive element meets
  44×44 (or the spaced-target exception). Sidebar nav items, filter
  chips, sortable column headers, etc.

### Non-EN README parity strategy

EN README is 585 lines. v1.17 brought non-EN to ~240 (41 %). v1.18
adds Why / Requirements / What-you-get / Configuration / Contributing
sections — non-EN now ~307 (53 %). Full 1:1 parity would require
translating the marketing-heavy "Quick start" walkthrough and the
"🌍 Getting Started" first-steps walkthrough; both are ~280 more
lines per locale × 7 = ~2000 lines. Deferred to v1.19+ as
marketing-translation work.

Practically, every non-marketing surface (architecture, API,
security, tests, a11y, limitations, license, requirements, features,
configuration, contributing) is now in the target language.

## Migration notes

- **Breaking**: any consumer hitting `/api/stream/scan-en` or
  `/api/stream/scan-ru` directly must migrate to
  `/api/stream/scan?source=ats|regional`. The Sunset header had been
  warning of this since v1.15.0.
- **No SPA changes required** by users: the SPA already moved to the
  consolidated endpoint in v1.16.
- **A11y change visible to keyboard users**: pressing Tab from page
  load now reveals a "Skip to main content" link before the sidebar.
  Mouse-only users see nothing changed.
