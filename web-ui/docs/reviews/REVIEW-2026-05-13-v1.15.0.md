## Build order rationale

Built in the order the FIX-PROMPT-FINAL.md spec recommended:

1. **PR-G** first — quick wins, removed the most visible carryover (the
   "EN or RU scan" literal still visible to every user with an empty
   /#/scan).
2. **PR-F** — docs-only, zero risk, no behavior change.
3. **PR-H** — biggest invisible-but-impactful fix: the v1.13.0 TSV SPA
   was unreachable from navigation because mode-page.js registered the
   same Router slug AFTER batch.js did. Renaming the legacy mode to
   `batch-prompt` resolved the collision.
4. **PR-E** — server-side schema acceptance had to land before front-end
   leverage. The summarizer is backward-compatible (legacy wins on
   collision).
5. **PR-D** — modes/_profile.md editor needed the schema acceptance
   from PR-E because the existence-check + scaffold flow needs to know
   what shape to expect.
6. **PR-B** — Legitimacy column is a thin polish that depended on the
   parser already extracting `r.legitimacy` (it has since v1.10).
7. **PR-A** — Block A-F realignment is documentation only (web-ui side);
   the prompt-emitter rewrite is in the parent which we can't touch.
8. **PR-C** — auto-pipeline is the biggest UX win and the last PR
   because it leverages every endpoint that came before (pipeline preview,
   evaluate, PDF inline, tracker). Client-side orchestrator over
   existing routes — no new server primitive needed.
9. **PR-I** — hero images last because the capture script needs the
   server running and the dashboard view to be in its v1.15 shape.

## Notable design decisions

### PR-A web-ui-only realignment (G-005)

The canonical career-ops.org rubric is A-F (Strategy/Personalization/STAR
stories at C/E/F). Our prompt still emits A-G (Risks/Verdict/Legitimacy).
The actual prompt template lives in `modes/oferta.md` in the PARENT —
forbidden territory per CLAUDE.md hard rule #1. So this PR:

- Documents the new canonical structure in all 8 help bundles + the
  `eval.subtitle` i18n key.
- Leaves the renderer block-letter-agnostic (no hardcoded `## G.`
  parsing anywhere — verified).
- Adds an explicit "Pre-v1.15 used A-G; we render those as-is for
  back-compat" callout in each help bundle.

This is safe to ship before the parent commit lands. Pre-v1.15 reports
written with A-G keep rendering unchanged. New A-F reports (after the
parent commit) will be styled identically because the renderer doesn't
care about block letters.

### PR-C client-side orchestrator (G-007)

The spec called for `POST /api/auto-pipeline` SSE. Implementing that
right would require refactoring helpers across 4 route modules (pipeline
preview, evaluate, PDF, tracker) so they can be called as functions
instead of HTTP. That's a substantial server refactor.

Client-side orchestrator achieves the same UX with much less risk:
chains the 4 existing HTTP endpoints, renders observable step-by-step
progress in a modal, extracts company/role/score heuristically from
the JD + evaluation markdown. Server-side SSE endpoint is documented
as v1.16+ follow-up.

### PR-D + PR-E coordinated schema work

Both PRs touch `server/lib/routes/content.mjs`. PR-E adds the
`summarizeProfile()` helper that accepts both schemas with legacy-wins
precedence. PR-D adds the `modes/_profile.md` CRUD routes. They share
no code path but ship together because they jointly fulfil
career-ops.org Quick Start §Step-3 and §Step-5 (the two most-edited
canonical files that previously had no UI).

## Test isolation regression (caught + fixed mid-PR)

While writing PR-E tests I discovered that the existing test pattern
(top-level `import { createApp }`) doesn't isolate `process.env.CAREER_OPS_ROOT`
because PATHS is evaluated at module-load time. The tests were
silently mutating the user's REAL `config/profile.yml` in the parent
project. This was a pre-existing latent bug.

Fix: switched both my new test files to the `before/after` + dynamic
`await import('../server/index.mjs')` pattern that `tests/batch-endpoints.test.mjs`
already used. Re-ran tests; verified parent `config/profile.yml` MD5
unchanged across `npm test` invocations.

**User impact:** if anyone ran an in-progress build between PR-E and
this fix, their `config/profile.yml` may have been overwritten with a
test stub. v1.15.0 ships only the fixed isolation pattern, so this
won't happen post-release.

## Migration notes

- **No breaking changes.** All new endpoints, new UI, additive
  i18n keys. Legacy profile.yml shapes continue to render
  identically; pre-v1.15 reports continue to render with their A-G
  block letters.
- **Deprecations** — `/api/stream/scan-en` + `scan-ru` now emit
  RFC 8594 Sunset headers (2026-10-01). Removal scheduled for
  v1.16.0. Consumers should migrate to `/api/stream/scan?source=ats|regional`.
- **Routing** — `/#/batch` now opens the v1.13.0 TSV SPA. Old
  bookmarks pointing at the mode-prompt builder are redirected to
  `/#/batch-prompt` (preserved through v1.15) with a deprecation
  banner. Scheduled for removal in v1.16.0.
- **Test isolation** — anyone running an in-progress dev build between
  the start of PR-E and the isolation fix may have had their parent
  `config/profile.yml` overwritten. Restore from your backup if you
  notice this; v1.15.0 ships the fix so no future release will repeat.
