## Notable design decisions

### Auto-pipeline server-side (#1, #2, #3, #8)

The v1.15.0 client-side orchestrator showed a generic spinner during
the 30–90 s Anthropic call. Moving server-side gives:
- **Real-time step progress** via SSE — the user sees `running` →
  `done` on each of 5 steps as it actually completes.
- **Curl-able CI smoke** — the entire pipeline can be exercised
  from a script: `curl -N -X POST /api/auto-pipeline -d '{"url":"…"}'`.
- **Report persistence** — `POST /api/reports` is the missing
  primitive that lets the orchestrator land report markdown in
  parent `reports/<slug>.md`. v1.15.0 dumped markdown inline in
  the modal and lost it on browser refresh.

The client-side orchestrator now drains the SSE stream and renders
deep-links to `#/reports/<slug>` + `#/tracker`. The heuristic
company/role/score/legitimacy extraction moved server-side
(removed ~80 lines of client logic).

### Slug path-traversal guard (#2)

The initial slug sanitizer left `..` from `../evil/slug` (caught by
the test fixture). Fixed by stripping leading `.` runs and
collapsing internal `...` sequences. `tests/reports-write.test.mjs::POST
/api/reports: slug sanitized` covers the regression.

### SmartRecruiters pagination (#4)

Stops on the FIRST of: empty page, `totalFound` reached, or 30-page
safety cap. The empty-page early-stop matters because some
SmartRecruiters tenants report a stale `totalFound` that's higher
than the actual count of open postings — without the early-stop
guard we'd loop the safety cap doing 28 pointless HTTPS calls.

### Workday graceful fallback (#7)

Module-level `lastWorkdayFallback` export gives the SPA's `#/scan`
Active Companies card a one-shot snapshot of the most recent
fallback reason. Wiring that into the UI chip rendering is v1.17+
work — the snapshot is plumbed but the chip isn't yet displayed.

### CI screenshot drift gate (#5)

The trickiest piece is that the parent doesn't exist on the CI
runner, so the workflow:
- builds a /tmp scaffold with the minimum files `/api/health`
  needs to be green;
- installs Playwright + chromium directly into the web-ui repo
  (via `npm install --no-save playwright`);
- boots the web-ui server with `CAREER_OPS_ROOT=/tmp/career-ops-fixture`.
This matches what `tests/batch-endpoints.test.mjs` already does in
the unit-test fixture pattern.

### i18n translation scope (#9, #10)

Translated v1.13.0 + v1.12.0 CHANGELOG entries in 6 locales (~12
substantial translations). Pre-v1.12 entries remain RU-bodied per
project convention; an i18n banner now explains this at the top of
each non-EN/non-RU CHANGELOG. Translating the full historical tail
(~1800 lines of v1.11.x / v1.10.x content × 6 locales) is deferred.

For READMEs, each non-EN file gets a ~35-line "v1.16.0 highlights"
section (auto-pipeline + curl example, all v1.16 features). Full
parity with the 585-line EN README is also deferred.

## Migration notes

- **No breaking API changes.** All new endpoints + new behaviors.
  Legacy `/api/stream/scan-{en,ru}` aliases still served with
  RFC 8594 Sunset headers (removal v1.17.0).
- **Cmd+K paste-URL UX change**: Enter now opens AutoPipeline.
  Shift+Enter preserves the legacy quick-add behavior. Old users
  who relied on Enter = quick-add should hold Shift.
- **Workday `strict:true` opt-in**: callers that depend on the
  v1.14 throw-on-4xx behavior can pass `{ strict: true }`.
- **Test fixture pattern**: new test files use `before/after +
  dynamic import` to honor `CAREER_OPS_ROOT`. Existing legacy
  tests still use the top-level-import pattern (works because
  they call `createApp` after their `before` hook). No external
  impact.
