## What changed

### `docs/career-ops-canonical.md` (new)

The single source-of-truth file. Eight sections:

1. **What career-ops is** — open-source job-search system, model-agnostic, runs as slash commands inside any AI CLI.
2. **Onboarding (first run)** — prereqs, clone, `cv.md`, `config/profile.yml`, `portals.yml`, optional LLM keys.
3. **Scanning job portals** — `portals.yml` 3-section schema, `npm run scan` vs `/career-ops scan`, output to `data/pipeline.md` + `data/scan-history.tsv`.
4. **Evaluating (single + batch)** — single via `oferta` mode; batch via `batch/batch-input.tsv` + `./batch/batch-runner.sh` with `--parallel` / `--min-score` / `--retry-failed`.
5. **Action thresholds by score** — ≥ 4.5 / 4.0–4.4 / 3.5–3.9 / < 3.5 with recommended next step each.
6. **Applying for a job** — `/career-ops apply <company>`, Playwright opens browser, system drafts numbered answers, user clicks Submit, then `Submitted.` flips tracker `Evaluated → Applied`.
7. **Playwright setup** — `npm install`, `npx playwright install chromium`, `claude mcp add playwright npx @playwright/mcp@latest`, `npm run doctor` to verify.
8. **career-ops vs career-ops-ui** — side-by-side table of which surface owns what (CLI owns form-fill, SPA owns CRM/browse).

### Help bundles — `docs/help/{en, ru, es, pt-BR, ko-KR, ja, zh-CN, zh-TW}.md`

Each gained the same front-matter block (translated per locale): About-career-ops summary, key concepts table, career-ops vs career-ops-ui table, score-thresholds table, link block to the 5 canonical guides. ### READMEs — `README*.md` (8 files)
