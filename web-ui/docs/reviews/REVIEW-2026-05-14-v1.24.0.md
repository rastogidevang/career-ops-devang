## Notable design decisions

### EN agent first, then 7 parallel translation agents — not 8 parallel from scratch

If we'd dispatched 8 parallel agents to fetch career-ops.org/docs URLs and write their own locale bundle, each agent would interpret the source slightly differently and the bundles would diverge in structure. Doing EN first establishes the canonical 16-H2 shape and deepening pattern; the 7 translation agents then mirror that EN shape section-for-section. Trade-off: ~5-10 min serial wait for the EN agent before the 7 parallel agents could start. Worth it for structural consistency.

### Deepen, don't bloat — 1113 → 1269 lines, not 2000

The "content-depth refresh" target was to integrate the five canonical guides into the existing 16 H2 sections, not to add new H2 sections. CI test `help-ui.test.mjs::section-parity` asserts exactly 16 H2 sections across all 8 locales — any drift would fail. The EN agent stayed comfortably within target (1113 → 1269 = +14% growth, mostly in the apply-checklist and concepts-inventory sections).

### Per-URL mention count enforced by CI

`canonical-docs-coverage.test.mjs` asserts each of the 5 URLs appears ≥ 2 times in the help bundle (front-matter reference + at least one CLI subsection). v1.24 over-delivers: `what-is-career-ops` × 4, `scan-job-portals` × 5, `apply-for-a-job` × 3, `batch-evaluate-offers` × 5, `set-up-playwright` × 3. Robust against a future single-mention regression.

### Scenario 31 — 6 of 6 sub-tests reachable via curl + grep

When I flagged scenario 31 as "needs browser agent + LLM credentials" in v1.23.0, the assumption was that visual verification of UI elements was the bottleneck. Re-reading the sub-tests, 6 of 6 turn out to be curl-reachable:

- 31.1 — score thresholds in help-bundle markdown (grep on file content)
- 31.2 — retired alias 404s (curl `-w '%{http_code}'`)
- 31.3 — `/api/apply-helper` checklist content (curl + grep)
- 31.4 — `/api/batch` endpoint shape (curl + JSON keys)
- 31.5 — `/api/health` Playwright check field (curl + JSON pluck)
- 31.6 — help-bundle URL coverage 5 × 8 matrix (curl per-locale + grep per-URL)

The visual sub-tests are spec'd as "open page X → see element Y" but every one already has a curl-reachable invariant. The browser-driven sub-tests remain valuable for catching UI-render regressions (icon glyphs, banner color, score-pill ::before glyphs) — those stay deferred to a future Claude Cowork run.

### RU CHANGELOG — retry agent finally landed

The v1.23.0 release shipped with the RU CHANGELOG retry agent in flight (its first dispatch had crashed with a socket-closed error after 13 min). The agent eventually completed during the v1.24 work window with a full 1542-line translation. v1.24 picks it up as a follow-up to the v1.23 M-9 work.
