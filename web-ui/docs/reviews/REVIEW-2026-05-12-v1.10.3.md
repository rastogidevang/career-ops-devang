## Findings closed

### F-019 — Express body-parser errors leaked HTML

**Where:** `server/index.mjs` — last middleware before the boot block.

**Problem.** `express.json({ limit: '5mb' })` and `express.raw({ limit: MAX_UPLOAD_BYTES })` throw `PayloadTooLargeError` / `SyntaxError` outside any route handler. The default Express handler returns `text/html` with a stack trace. The SPA's `try { await res.json() }` then throws "Unexpected token" and shows a generic "fetch failed" toast.

**Fix.** Added a 4-arg `app.use((err, req, res, _next) => …)` middleware (must be LAST). Returns:
- `413 { error, limit, length }` for `entity.too.large` / `LIMIT_FILE_SIZE`.
- `400 { error: "malformed request body" }` for `entity.parse.failed` / `charset.unsupported`.
- `400 { error: "malformed JSON body" }` for stray `SyntaxError` instances.
- `500 { error: "internal server error" }` last-resort, with the real `err` logged server-side via `console.error`.

**Tests:** `tests/global-error-handler.test.mjs` — malformed JSON to `/api/pipeline` → 400 JSON; 11 MB upload to `/api/cv/import` → 413 JSON.

### F-015 — `📄 Generate PDF` only worked on `#/cv`

**Where:** `server/lib/routes/runners.mjs` (3 new SSE endpoints) + `public/js/lib/pdf-generate.js` (new shared helper) + `public/js/views/reports.js`, `evaluate.js`, `deep.js`.

**Fix.**
- New `streamPdfFor({ res, slug, markdown, title, format })` helper inside `runners.mjs` — writes `output/<slug>-input-<TS>.html` and spawns `generate-pdf.mjs <input> <output> --format=a4`. Slugs are sanitized to `[\w-]+`.
- `GET /api/stream/pdf/report?slug=` — reads `reports/<slug>.md`. 404 when missing.
- `GET /api/stream/pdf/deep?name=` — reads `interview-prep/<name>` (must end with `.md`). 400 / 404 on bad name.
- `POST /api/stream/pdf/inline { markdown, title?, slug? }` — for unsaved content (evaluate prompt, deep prompt before persist).
- `public/js/lib/pdf-generate.js` — single helper `PdfGenerate.run({ kind, ... })` covering all four kinds (`cv | report | deep | inline`). Snapshots latest PDF in `output/` before stream, triggers `<a download>` on `done` when a new file appeared. Inline kind uses `fetch + ReadableStream` to consume SSE from a POST.
- Wired into `reports.js` (single-report header), `evaluate.js` (manual and live result cards), `deep.js` (every `showResult` card — uses `kind: deep` when `opts.saved` exists, falls back to `inline` for unsaved live runs).
- New i18n key `common.generatePdf` across 8 locales.

**Tests:** `tests/pdf-extra-routes.test.mjs` — 5 cases proving each endpoint spawns `generate-pdf.mjs` with the right 3 positional args + 404 / 400 reject paths.

### F-013 — `HH_USER_AGENT` exposed as a top-level setting

**Where:** `server/lib/env-config.mjs::KEY_GROUPS` (new export) + `server/lib/routes/config.mjs` (new `groups` + `regionalActive` in response) + `public/js/views/config.js::renderGroup`.

**Fix.**
- `KEY_GROUPS` map classifies every known key as `core | runtime | regional`. `HH_USER_AGENT` is the only `regional` entry today.

**Tests:** `tests/config-groups.test.mjs` — `/api/config` exposes `groups`, classification correct, `regionalActive` flips on when portals.yml gains a source.
