## Notable design decisions

### M-4 — Entity-decode as a soft pre-pass, not a full sanitizer

`stripDangerousMarkdown` is documented as defense-in-depth — the real XSS guard is the client-side `UI.md` escape-first pipeline. v1.22 hardens the at-rest CV file so consumers that bypass `UI.md` (cat, grep, third-party tools) don't get a live script back. We deliberately decode ONLY four named entities (`&lt;`, `&gt;`, `&amp;`, `&quot;`) plus numeric/hex entities — NOT every named entity. Decoding `&copy;` to `©` would change unrelated text in stored CVs. The test sweep ([`tests/cv-xss-bypasses.test.mjs`](../../tests/cv-xss-bypasses.test.mjs)) covers entity-encoded `<script>`, `javascript:`, `onerror=`, SVG-onload via data URI, and double-encoded `&amp;lt;…`.

### M-6 — `setTimeout` chain, not `setInterval`, for backoff

`setInterval(fn, ms)` doesn't pick up the new delay when the interval changes — each fire uses the originally-set ms. To implement true exponential backoff (3 s → 6 s → 12 s → 24 s → 60 s), each step must `setTimeout` itself with the current delay. The new pattern lives in `public/js/api.js:22-48`. Resets to 3 s on the first 2xx recovery.

### M-3 — `::before` glyph instead of touching every render site

Score pills are rendered in 3 view files (`tracker.js`, `reports.js`, `dashboard.js`). Adding the glyph at every render site would mean 6+ edits with potential drift. Putting it on `.score-{high,mid,low}::before` in CSS gives the redundant cue everywhere at once — pure CSS hardening, render code untouched. The `IDE` flagged `speak: none` (Firefox/Safari don't support); we dropped it. Screen-reader users already get the numeric score read aloud; the glyph is for visual users in monochrome / high-contrast modes.

### M-1 — Hint copy lookups at render time, not at mount

Each mode-page field's `i18n.hint` key is resolved through `t()` at render time, not pre-bound to the spec. This means a locale switch re-renders both the label and the hint without a re-render of the entire view. The `aria-describedby="…-hint"` wire stays stable across language switches.

### `safe-fetch.mjs` test transport — `_setTransport(fn)` not `globalThis.fetch` mock

Pre-v1.21 SSRF tests mocked `globalThis.fetch`. The v1.21 rewrite uses `node:http(s)` directly, bypassing fetch. The pipeline-preview test in `tests/pipeline-preview.test.mjs` was rewired to use the `_setTransport` injection point — pre-pinned-IP layer, so DNS rebind logic still applies but no real socket is opened. v1.22's M-2 cap test reuses the same pattern.

### Translation quality: dispatch parallel agents per locale, not one big translator

The user explicitly flagged "корявые слова" in the Russian translation. Rather than fix RU and let the other 6 drift, we dispatched 7 parallel translation-quality agents — one per non-EN locale — each instructed to:

1. Match EN structure end-to-end (sections, tables, code blocks in same order).
2. Apply native technical style (no calques like "функционал" → use "функциональность" / "возможности").
3. Refresh v1.21/v1.22 surface mentions.
4. Bump release + test badges.
5. Preserve code blocks, URLs, file paths, shields.io badges verbatim.

Final line counts: 569 (zh-CN) to 603 (ru). EN master is 588.
