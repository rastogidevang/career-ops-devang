## What changed

### PR-4 multer pipeline (`server/lib/routes/content.mjs` + `package.json`)

Added `multer ^2.1.1` as a dependency. The CV import route dispatches on Content-Type:
- `multipart/form-data` → `multer().any()` extracts the first file part; field name is irrelevant.
- Anything else → original `express.raw({ type: '*/*' })` path. X-Filename header gives the extension hint.

Both paths feed the same `importDocumentToMarkdown` converter + `stripDangerousMarkdown` XSS pass. Conversion errors come back as 422; payload-too-large as 413 (caught by the F-019 global handler).

Tests rewrote the v1.10.2 "multipart returns 415" assertion into a "multipart parsed properly" assertion. The no-side-effect-on-`cv.md` invariant is preserved (the route returns the converted markdown; persisting still requires an explicit `PUT /api/cv`).

### Adapter registry (`server/lib/portals/adapters/*.mjs` + `registry.mjs`)

Three new adapter modules — one per existing ATS:
- `greenhouse.mjs` — matches `job-boards(.eu)?.greenhouse.io/<slug>` or explicit `api:` field; endpoint `boards-api.greenhouse.io/v1/boards/<slug>/jobs`.
- `ashby.mjs` — matches `jobs.ashbyhq.com/<slug>`; endpoint includes `?includeCompensation=true`.
- `lever.mjs` — matches `jobs.lever.co/<slug>`; endpoint `api.lever.co/v0/postings/<slug>`.

Each exports the same contract: `{ id, label, matches(company), buildEndpoint(company), fetch(endpoint, opts) }`. The HTTP fetchers in `server/lib/sources/` stay where they were — adapters are thin wrappers that expose the uniform contract.

`registry.mjs` exports `ALL_ADAPTERS` and `resolveAdapter(company) → { adapter, endpoint } | null`. `en-scanner.mjs::detectApi()` now delegates to the registry; the legacy return shape (`{ type, url }`) is preserved so external callers keep working. `FETCHERS` table sourced from `Object.fromEntries(ALL_ADAPTERS.map(a => [a.id, a.fetch]))` — any new adapter automatically becomes callable.

### Batch evaluate (`server/lib/routes/batch.mjs` + `public/js/views/batch.js`)

Four new endpoints:
- `GET /api/batch` → `{ exists, runnerExists, raw, rows, additions }`
- `PUT /api/batch { raw }` — write `batch/batch-input.tsv`. 413 on > 1 MB. 400 when no row contains a URL.
- `GET /api/stream/batch[?parallel=N][&minScore=4.0][&dryRun=1][&retry=1]` — SSE wrapping `bash batch/batch-runner.sh`. Emits `start`, `log`, `done`, `error` events. SIGTERMs the child on client disconnect.
- `POST /api/batch/merge { dryRun? }` — runs `node merge-tracker.mjs`.

`PATHS` gained `batchDir`, `batchInput`, `batchRunner`, `batchAdditionsDir`.

SPA view at `#/batch` — TSV editor pane, run controls (parallel select, min-score input, dry-run + retry checkboxes), live SSE console, pending-additions list with one-click Merge button. Sidebar link under "Decision" group. 21 new i18n keys × 8 locales. Warning card when `batch/batch-runner.sh` is missing from the parent project.

### Locale-aware mode scaffolding (`server/lib/prompts.mjs`)

New `SCAFFOLD_STRINGS` map: `readFiles`, `userContext`, `modeRoleLine`, `evalRoleLine`, `modeTemplate` × 8 locales. New `scaffold(key, lang)` exported helper.

`buildModePrompt` and `buildEvaluationPrompt` now read scaffolding from this map. The parent's `modes/<slug>.md` body is still inlined verbatim (read-only per CLAUDE.md hard rule #1), but the wrapper text around it gets localized.

### Dark-theme table fix (`public/css/app.css`)

`table.tbl tr:hover td { background: #fafafa }` → `var(--beach)`. `.tab-btn` lost its hardcoded `#fff` / `#f7f7f7` / `#d8d8d8` fallbacks in favor of `var(--paper)` / `var(--beach)` / `var(--slate)`. The dark palette now reaches every hover state.

Bonus: `.row-boosted` accent strip class — pairs with `_boosted` server-side flag in scan results, but works on both light and dark themes.
