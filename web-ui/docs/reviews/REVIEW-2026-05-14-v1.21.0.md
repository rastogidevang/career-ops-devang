## Notable design decisions

### B-1 — `safe-fetch.mjs` over undici Agent

The textbook fix for DNS-rebind TOCTOU is `undici.Agent` with a custom
`lookup` option that returns the pre-validated IP. Node 18+ ships
undici as the underlying `fetch` implementation but does not directly
expose `import { Agent } from 'undici'` — that would require adding
`undici` as a runtime dep, contradicting CLAUDE.md's "no new runtime
deps lightly" rule (current production deps: `express`, `js-yaml`,
`multer`).

The chosen alternative — `safe-fetch.mjs` — uses Node built-in
`node:http` / `node:https` with `host` + `hostname` set to the
pre-validated address and `servername` set to the original hostname so
TLS SNI / cert validation still target the canonical host. Same
security guarantee, no new dep, ~200 LOC.

Tests inject stubs via `_setTransport(fn)` instead of monkey-patching
`globalThis.fetch` — the existing `tests/pipeline-preview.test.mjs`
needed a 30-line rewrite to use the new injection point because the
SSRF path no longer goes through fetch.

### H-4 — `sanitizePathName` returns empty string, callers must 400

The previous bare regex (`replace(/[^\w\-.]/g, '')`) silently swallowed
all-dot inputs (`..pdf` → `..pdf`, `....md` → `....md`) and let them
through to `existsSync` checks that returned 404 — superficially fine
but the *contract* was leaky: callers had no way to distinguish "valid
name that doesn't exist" from "malicious name that got rewritten to
something else." `sanitizePathName` now returns `''` for any input
that loses all useful characters; every caller checks for the empty
string and returns 400. The 12 path-traversal tests sweep
`..`, `....pdf`, `../../etc/passwd`, URL-encoded `%2e%2e`, NUL bytes,
and the legacy "name with valid suffix" pattern.

### H-5 — rate limit only on public bind, opt-in via env

Loopback users see no behavioural change. The middleware checks
`isPubliclyExposed()` (returns true when `HOST !== 127.0.0.1 / ::1 /
localhost`) and only consumes a bucket slot on public bind. Default
is 10 req/min/IP, configurable via `LLM_RATE_LIMIT="N/Ws"`. Returns
429 with `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Reset`
headers so browsers can back off automatically. Bridge to v2.0 P-12
auth gate — when proper per-user accounting lands, this middleware
goes away.

### H-6 — in-memory mutex, not OS file lock

Express is single-process; the race exists between concurrent
`async` handlers within one Node process, not across processes. An
in-memory mutex (`new Map<path, Promise>` chain) is sufficient and
adds no I/O. If/when the server scales horizontally (v2.x topic),
this becomes a Redis lock or an OS-level `fcntl` flock. Documented
in the module header so the assumption is visible.

### Why no integration test for pipeline.mjs concurrent writes

The unit test `tests/concurrent-tracker-write.test.mjs` covers
both the `withFileLock` primitive (4 cases) and a 20-concurrent
end-to-end POST `/api/tracker`. The pipeline-side flow uses the
same primitive — fully covered transitively. Adding a second
integration test that re-runs the same fixture against
`/api/pipeline` would add ~80 LOC of duplicated setup for no
new signal. (An earlier draft of the test file had this; it
exposed a separate issue with module-graph caching across dynamic
imports that's unrelated to the lock contract. Dropped.)

### H-3 — extending i18n-coverage.test, not replacing it

The existing `tests/i18n-coverage.test.mjs` already validated that
every DICT key carries all 8 locales. v1.21.0 adds the inverse
direction: every `t('key', …)` call in `public/js/**/*.js` must map
to a DICT entry. Without the inverse direction, you can drift in
either way — old keys could go orphaned, new keys could appear
without translation. Now CI catches both.
