## What changed

### New adapters (`server/lib/portals/adapters/{workable,smartrecruiters,workday}.mjs`)

Each follows the v1.13.0 uniform contract `{ id, label, matches(company), buildEndpoint(company), fetch(endpoint, opts) }`. Detection is purely from `company.careers_url`; an explicit `company.api` field overrides URL detection (same priority chain as v1.13.0 adapters).

- **Workable** — URL patterns: `apply.workable.com/<slug>` OR legacy `<subdomain>.workable.com`. Endpoint: `https://apply.workable.com/api/v3/accounts/<slug>/jobs?details=true`. Public REST, no auth.
- **SmartRecruiters** — URL patterns: `jobs.smartrecruiters.com/<slug>` OR `careers.smartrecruiters.com/<slug>`. Endpoint: `https://api.smartrecruiters.com/v1/companies/<slug>/postings`. Public REST, first page only (limit=100); pagination is a v1.15+ follow-up.
- **Workday (beta)** — URL pattern: `<tenant>.wd<N>.myworkdayjobs.com(/<lang>)?(/<site>)?`. Endpoint: `POST https://<tenant>.wd<N>.myworkdayjobs.com/wday/cxs/<tenant>/<site>/jobs` with body `{ appliedFacets: {}, limit: 100, offset: 0, searchText: "" }`. Defaults `site=External` when the careers_url has no site path (covers the majority of public Workday tenants). Beta because (a) some tenants gate CXS behind CAPTCHA — adapter throws in that case, and (b) tenant + site discovery is per-customer.

### New HTTP sources (`server/lib/sources/{workable,smartrecruiters,workday}.mjs`)

Raw `fetch()` to the ATS endpoint, then normalize the response array to the canonical job shape `{ id, title, company, url, location, isRemote, workplaceType, source, … }` that the rest of the pipeline consumes. `source` field is set per-adapter id (`"workable"` / `"smartrecruiters"` / `"workday"`) so the chip filter on `#/scan` segregates them automatically. No persistence layer changes were needed — `runEnScan` writes `data/pipeline.md` and `data/scan-history.tsv` exactly as before.

### Registry (`server/lib/portals/registry.mjs`)

Three imports + three entries in `ALL_ADAPTERS`. That's it — v1.13.0's design intent was "adding a new ATS = one file, no scanner changes." Verified: `en-scanner.mjs::detectApi()` and `FETCHERS` table both pick up the new adapters via `Object.fromEntries(ALL_ADAPTERS.map(a => [a.id, a.fetch]))` without any edits.
